## Reflection on Commits and Tasks

Madhusudhan made approximately 3 commits, while Abhiram made approximately 3 commits. The number of commits was related to the type of work completed. There were fewer commits in some technical configuration work as several screenshots were taken in a session, and more commits, but smaller in size, when documenting and updating diagrams. Both students shared in all aspects of the project such as set up, testing/firewall, hardening, traffic analysis, risk assessments, and report writing. Commitments were almost exclusively made in Weeks 6-10 and this was adequate since the project work was done sequentially and not at the end.

## Reflection on Group Work

The team collaborated through breaking up the technical and documentation tasks. Using WhatsApp and meetings, communication was done to determine issues regarding VirtualBox adapter order, HTTP caching during packet capture, and firewall zone configuration. These problems were addressed through command testing, comparing screenshots and verifying each other's work. For future group projects, it is better to allocate the tasks earlier, commit regularly to the GitHub and have a checklist to avoid delays and have students contributing equally.
