# Project Plan

## Communication Plan

The group will be communicating through Microsoft Teams and WhatsApp twice a week. They will have a meeting every Wednesday to review progress and a second check-in Sunday to make sure tasks have been completed and to plan next week's progress. The students will use GitHub as the version control system and each student will commit their own screenshots, diagrams, packet captures, report writing, or spreadsheet updates. Should there be any technical problem, or contribution missing, or anything is delayed, it will be discussed immediately and the project will not be delayed. Final files will be checked by both students prior to sending.

## Schedule

The plan of tasks for each group member for the remainder of the project is:

* Week 5: Organize into groups, establish github repository, discuss how to communicate, and develop initial schedule. Students read through the specification.

* Week 6: Madhusudhan sets up the OpenWRT VM, checks `ip addr`, `ip route`, and VirtualBox adapter settings. Abhiram records Windows host IP details and starts the lab network diagram.

* Week 7: Madhusudhan develops and tests the business website on OpenWRT. Abhiram finishes the lab and production network diagrams and IP allocation table.

* Week 8: Madhusudhan sets up the block/allow rules for both ICMP and HTTP. Abhiram sets up SSH port change and management port 81 restriction. Both collect screenshots.

* Week 9: Madhusudhan completes password change, `/etc/shadow` review, and HTTP traffic capture. Abhiram completes SSH key authentication, disables `odhcpd`, and captures SSH traffic.

* Week 10: The risk assessment, security controls, project reflection, final Markdown report, PDF exports, repository ZIP and recorded demonstration video are completed by both students.
